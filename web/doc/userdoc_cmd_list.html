You have now seen how to use a few of the CLI commands (in the previous section). In this section we describe all of the available, valid commands that you can use, and how to properly use them. For the effect that each command has with regards to editing an image, please see the respective tool in the GUI Tools section.

Proper Command Usage

Most commands in the CLI require an input and output image to operate on and output to, respectively. The only command that does not require an input/output image is the help (-h) command.

Filter commands can be called back-to-back in the command line, as long as they are all between one input file and one output file. An example would be:
<program-path> <img.in> -sharpen 5.3 -edge <img.out>
The only exception is the compare filter, which only operates on an input and an output file, and will fail if used with other commands.

In all examples of Supported Commands below, we say <program-path> to mean the path where the program executable is located. When in the Image Processing Suite, this is ./build/bin/MIA. We say <img.in> to mean a valid input image, and <img.out> to mean a valid output image.

Input and Output Files

  • Input File
    The input file must be an existing file of valid type (either .jpeg/.jpg or .png), and it must be entered with a path relative to the root directory of the application (as in the example in the previous section). If there is no input file specified, or if it is invalid, then an error message will be shown.

  • Output File
    The output file must be named with a valid type into an existing directory (if the directory does not exist the file will not be output). If there is no output file specified, or if it is invalid, then an error message will be shown. Again, the output directory is relative to the root directory of the Image Processing Suite.

  • Image Stacks
    Operating on a series of images (a "stack") is possible with the CLI. The syntax for doing so is:
    <img###.in> <commands> <img###.out>
    The three pound signs represent a three-digit number, which is the file where the application will begin to edit the files. The application will step through all files one at a time until it runs out of images. Be careful that your images are all numbered successively and do not skip any numbers! This would cause the application to stop at the break in the sequence, leaving all the higher-numbered files after the break unedited.

    Also, a note about the numbering: The numbers for the input and output files do not necessarily have to match. If they do not, the application will simply start with the output number you give it and increment by one for every input file it alters. Thus you could input a start file numbered 057 and an output file numbered 0, and if the program edited 30 files then the images from input files 57 through 86 would be output to files 0 through 29.

Supported Commands

  • Help: -h
    This command will display a message in the command line/terminal describing what commands exist and how to properly use them. Example:
    <program-path> -h

  • Sharpen: -sharpen <float>
    Sharpens the input image(s) by the specified amount, and conforms to the GUI Tool's input range restrictions. This command requires a floating point number to be entered immediately after it in the command line, otherwise an error message will appear. Examples:

    <program-path> <img.in> -sharpen 5.3 <img.out>

    Will have no effect but is valid:
    <program-path> <img.in> -sharpen 0.0 <img.out>

    (Invalid Input! Range: 0 - 100)
    <program-path> <img.in> -sharpen 101.0 <img.out>

    (Invalid Input! Requires a floating point number)
    <program-path> <img.in> -sharpen <img.out>

  • Edge Detect: -edge
    Performs edge detection on the input image confomring to its GUI Tool. Examples:

    <program-path> <img.in> -edge <img.out>

  • Threshold: -threshold <float>
    Applys the threshold filter to the image, with the GUI Tool's same restrictions and functionality. This command requires a floating point number to be entered immediately after it in the command line, otherwise an error message will appear. Examples:

    <program-path> <img.in> -threshold 0.4 <img.out>

    (Invalid Input! Range: 0.0 - 1.0)
    <program-path> <img.in> -threshold 1.1 <img.out>

    (Invalid Input! Requires a floating point number)
    <program-path> <img.in> -threshold <img.out>

  • Quantize: -quantize <int>
    Applys the quantize filter to the image, with the GUI Tool's same restrictions and functionality. This command requires an integer to be entered immediately after it in the command line, otherwise an error message will appear. Examples:

    <program-path> <img.in> -quantize 8 <img.out>

    (Invalid Input! Range: 2 - 256)
    <program-path> <img.in> -quantize 1 <img.out>

    (Invalid Input! Requires an integer)
    <program-path> <img.in> -quantize <img.out>

  • Blur: -blur <float>
    Applys the quantize filter to the image, with the GUI Tool's same restrictions and functionality. This command requires a floating point number to be entered immediately after it in the command line, otherwise an error message will appear. Examples:

    <program-path> <img.in> -blur 6.8 <img.out>

    (Invalid Input! Range: 0.0 - 20.0)
    <program-path> <img.in> -blur 20.4 <img.out>

    (Invalid Input! Requires a floating point number)
    <program-path> <img.in> -blur <img.out>

  • Saturate: -saturate <float>
    Increases, decreases, or inverts the colors of the image. Has an input range of -10.0 to 10.0. This command requires a floating point number to be entered immediately after it in the command line, otherwise an error message will appear. Examples:

    <program-path> <img.in> -saturate 6.8 <img.out>

    (Invalid Input! Range: -10.0 to 10.0)
    <program-path> <img.in> -saturate 12.4 <img.out>

    (Invalid Input! Requires a floating point number)
    <program-path> <img.in> -saturate <img.out>

  • Channel: -channel <float> <float> <float>
    Applys the channel filter to the image, with the GUI Tool's same restrictions and functionality. This command requires three floating point numbers to be entered immediately after it in the command line, otherwise an error message will appear. Examples:

    <program-path> <img.in> -channel 6.8 0.4 2.3 <img.out>

    (Invalid Input! Range: 0.0 - 10.0)
    <program-path> <img.in> -channel 20.4 7.2 0.1 <img.out>

    (Invalid Input! Requires three floating point numbers)
    <program-path> <img.in> -channel <img.out>

  • Compare: -compare
    Compares two files and outputs 1 if they are exactly identical, or 0 if they are not. Can have no other commands entered with it. Examples:

    <program-path> <img.in> -compare <img.out>

    (Invalid Input! Compare cannot be entered with another command)
    <program-path> <img.in> -compare -sharpen 4.1 <img.out>

  • Convert File Type: <img.in> <img.out>
    Takes an image of valid type and converts it to another valid type. Works with other commands as well, taking one image type, modifying it, and outputting it as another image type. Examples:

    <program-path> <img.jpeg> <img.png>

    <program-path> <img.png> -sharpen 2.82 <img.jpg>

    (Invalid Input! Gif file extension not supported)
    <program-path> <img.png> <img.gif>
