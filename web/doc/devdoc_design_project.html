Developer Resources - Image Processing Suite

Project Design

src
src contains the entirety of the source code for the Image Processing Suite.

app
app contains the applications that implement the core image processing functionality. Since new applications may be built in the future, this directory shall serve to encompass all application build directories. FlashPhoto and MIA are the two applications that have been built into the Image Processing Suite.

FlashPhoto
FlashPhoto contains the FlashPhoto application source code files, as well as a Makefile for compiling and linking files.

src
src contains the .cc files required to instantiate the graphics editor/photo manipulation application, as well as the corresponding .h files. Since the Makefile works recursively through a directory, it's necessary to keep the prerequisites self-contained.

MIA
MIA contains the directory containing the MIA application source code files, as well as the Makefile for compiling and linking files.

src
src contains the .cc files required to instantiate the graphical and command-line medical imaging application, as well as the corresponding .h files. Since the Makefile works recursively through a directory, it's necessary to keep the prerequisites self-contained.

lib
lib contains libraries that are capable of being shared by multiple applications. Since new libraries may be added in the future, this directory shall serve to encompass all shared libraries. libimgtools is the currently available shared library.

libimgtools
libimgtools contains everything required to extend the image processing functionality in order to build a new application. This includes all the required .cc files and corresponding .h files, as well as a Makefile for compiling and linking.

src
src contains the .cc files for building an instance of the imaging software, as well as a directory with the corresponding .h files. Since the Makefile works recursively through a directory, it's necessary to keep the prerequisites self-contained.

include
include contains the .h files corresponding to the appropriately named .cc files in src. Due to the large number of source code files within src, this directory helps keep the files organized.
